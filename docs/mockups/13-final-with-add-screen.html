Design Decisions

  • Weekly progress dots moved to right side of habit row for cleaner visual hierarchy - name on left, progress on right
  • Add Habit uses a bottom sheet modal pattern for quick, low-friction input while maintaining context
  • Frequency toggle is simple binary choice - Daily or X times per week
  • Weekly picker shows tappable number buttons (1-7) that are easy to hit on mobile
  • Save button is disabled until habit name is entered, providing clear validation feedback
  • Modal can be dismissed via X button or swipe down (implied by drag handle)
  • Form is intentionally minimal - just name and frequency, nothing else to slow users down
